Why It Is Still a Stub

The frontend path is ready, but the actual launch still depends on a live gameplay backend at play-vaultfront.vaultsparkstudios.com plus its companion API and websocket infrastructure. Until that runtime is established, the public URL stays on this project page instead of exposing a half-live client.

Launch Checklist

  • Provision the shared VPS runtime stack
  • Deploy `vaultfront-play` and `vaultfront-api` services
  • Configure DNS and TLS for `play-vaultfront` and `api-vaultfront`
  • Verify health checks, websocket routing, and CORS from the public path
  • Only then replace this stub with the real playable client rollout
